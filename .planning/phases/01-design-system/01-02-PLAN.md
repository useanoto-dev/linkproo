**Step 3 — Type rowToSmartLink parameter (per D-08):**

Change the function signature from:
```typescript
export function rowToSmartLink(row: any, viewCount = 0, clickCount = 0, ownerPlan?: string): SmartLink {
```
to:
```typescript
export function rowToSmartLink(row: Partial<SmartLinkRow>, viewCount = 0, clickCount = 0, ownerPlan?: string): SmartLink {
```

The function body accesses fields like `row.id`, `row.slug`, `row.business_name`, etc. With `Partial<SmartLinkRow>`, all fields become `T | undefined`. The existing defensive patterns (`|| ""`, `?? undefined`, etc.) already handle this correctly. If TypeScript errors appear on specific lines (e.g., casting `row.buttons` or `row.hero_focal_point`), the existing cast patterns like `(row.buttons as SmartLink["buttons"]) || []` remain valid because TypeScript accepts casts from `unknown | undefined` to a specific type.
