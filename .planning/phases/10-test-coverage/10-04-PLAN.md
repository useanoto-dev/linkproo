For remove and reorder: test the store.updateLink pattern directly (since remove/reorder live inside SortableList.tsx as inline callbacks — testing them via store operations covers the same data path).

Import:
```typescript
import { describe, it, expect, vi, beforeEach } from 'vitest';
import { renderHook, act } from '@testing-library/react';
import { useBlockOperations } from '@/hooks/use-block-operations';
import { useEditorStore } from '@/stores/editor-store';
import type { SmartLink } from '@/types/smart-link';
```

BASE_LINK constant (SmartLink with empty blocks/buttons):
```typescript
const BASE_LINK: SmartLink = {
  id: 'link-1', slug: 'test', businessName: 'Test', tagline: '',
  heroImage: '', logoUrl: '', backgroundColor: 'from-gray-50 to-white',
  textColor: 'text-white', accentColor: '#f59e0b',
  buttons: [], pages: [], badges: [], floatingEmojis: [], blocks: [],
  views: 0, clicks: 0, isActive: true, createdAt: '2026-01-01T00:00:00.000Z',
};
```

beforeEach: reset the store with `useEditorStore.getState().resetLink(BASE_LINK)`.

Helper to get hook with live store callbacks:
```typescript
function getHook(link: SmartLink) {
  const { updateLink, setLink } = useEditorStore.getState();
  return renderHook(() =>
    useBlockOperations({ link, updateLink, setLink })
  );
}
```

**Test group: addBlock**

a) "addBlock('text') appends a text block to link.blocks"
   - `const { result } = getHook(BASE_LINK)`
   - `act(() => result.current.addBlock('text'))`
   - `const state = useEditorStore.getState()`
   - `expect(state.link.blocks).toHaveLength(1)`
   - `expect(state.link.blocks[0].type).toBe('text')`

b) "addBlock('button') appends to link.buttons not link.blocks"
   - `getHook(BASE_LINK)` then `act(() => result.current.addBlock('button'))`
   - `expect(state.link.buttons).toHaveLength(1)`
   - `expect(state.link.blocks).toHaveLength(0)`

c) "addBlock assigns order = 0 when list is empty"
   - After addBlock('text'): `expect(state.link.blocks[0].order).toBe(0)`

d) "addBlock assigns incrementing order for second block"
   - Use link with one existing block at order 0
   - `const linkWith1 = { ...BASE_LINK, blocks: [{ id: 'b1', type: 'text' as const, order: 0, text: '', textSize: 'md', alignment: 'left', color: '#fff' }] }`
   - `getHook(linkWith1)` then `act(() => result.current.addBlock('text'))`
   - `const blocks = useEditorStore.getState().link.blocks`
   - `expect(blocks).toHaveLength(2)`
   - `expect(blocks[1].order).toBe(1)` (next after max=0)

e) "addBlock triggers toast.success"
   - `import { toast } from 'sonner'`
   - After addBlock: `expect(vi.mocked(toast.success)).toHaveBeenCalledOnce()`

**Test group: insertBlockAt**

f) "insertBlockAt(0) bumps existing block from order 0 to order 1"
   - `const linkWith1 = { ...BASE_LINK, blocks: [{ id: 'b1', type: 'text' as const, order: 0, text: '', textSize: 'md', alignment: 'left', color: '#fff' }] }`
   - `getHook(linkWith1)` then `act(() => result.current.insertBlockAt('header', 0))`
   - `const blocks = useEditorStore.getState().link.blocks`
   - `expect(blocks).toHaveLength(2)`
   - New block at order 0: `expect(blocks.find(b => b.type === 'header')?.order).toBe(0)`
   - Existing block bumped: `expect(blocks.find(b => b.id === 'b1')?.order).toBe(1)`

g) "insertBlockAt with type='button' inserts into buttons array"
   - `getHook(BASE_LINK)` then `act(() => result.current.insertBlockAt('button', 0))`
   - `expect(useEditorStore.getState().link.buttons).toHaveLength(1)`

**Test group: remove block (store.updateLink pattern)**

h) "removing a block by filtering it out updates the store"
   - Reset store to link with 2 blocks: `[{ id: 'b1', order: 0, ... }, { id: 'b2', order: 1, ... }]`
   - `const { updateLink } = useEditorStore.getState()`
   - `act(() => updateLink({ blocks: link.blocks.filter(b => b.id !== 'b1') }))`
   - `expect(useEditorStore.getState().link.blocks).toHaveLength(1)`
   - `expect(useEditorStore.getState().link.blocks[0].id).toBe('b2')`

**Test group: reorder blocks (store.updateLink with reassigned orders)**

i) "reordering blocks updates order properties correctly"
   - Reset store to link with 3 blocks at orders 0, 1, 2

   - `act(() => updateLink({ blocks: reordered }))`
   - `expect(useEditorStore.getState().link.blocks[0].order).toBe(0)`
   - `expect(useEditorStore.getState().link.blocks[0].id).toBe(blocks[2].id)` (formerly last, now first)

j) "reordering creates a history entry allowing undo"
   - After reorder, `expect(useEditorStore.getState().canUndo).toBe(true)`
   - After undo: `expect(useEditorStore.getState().link.blocks[0].id).toBe(blocks[0].id)` (original order restored)
