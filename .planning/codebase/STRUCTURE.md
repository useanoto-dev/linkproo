## Directory Purposes

**`src/pages/`:**
- Purpose: One file per route; pages orchestrate hooks and compose layout + feature components
- Contains: Page components only — no business logic, no direct Supabase calls
- Key files: `LinkEditor.tsx` (main editor page, 806 lines), `PublicLinkPage.tsx` (public renderer + SEO)

**`src/pages/admin/`:**
- Purpose: Admin panel pages, each guarded by `AdminRoute`
- Contains: `AdminDashboardPage.tsx`, `AdminUsersPage.tsx`, `AdminAnalyticsPage.tsx`, `AdminVideoaulasPage.tsx`, `AdminSupportPage.tsx`

**`src/components/editor/`:**
- Purpose: All UI panels and tools used within the link editor
- Contains: `BlockEditor.tsx` (1944 lines — block list, drag-and-drop, per-block forms), `ElementsSidebar.tsx`, `ThemePanel.tsx`, `EffectsPanel.tsx`, `SubPageEditor.tsx`, `DeviceFrame.tsx`, `ImageUploader.tsx`, `ButtonBlockEditor.tsx`, `CropEditor.tsx`

**`src/components/preview/`:**
- Purpose: Read-only rendering components for SmartLink display (shared between editor preview and public page)
- Contains: `BlockRenderer.tsx` (748 lines), `ButtonPreview.tsx`, `AnimatedButtonBlock.tsx`, `CountdownBlock.tsx`, `FaqAccordionItem.tsx`, `FloatingEmoji.tsx`, `preview-utils.ts`

**`src/components/ui/`:**
- Purpose: shadcn/ui primitive components — buttons, inputs, dialogs, cards, etc.
- Contains: ~40 generated component files
- Committed: Yes. Generated by shadcn CLI but treated as owned source.
- Do not manually edit; re-generate via `npx shadcn-ui@latest add <component>` if updating

**`src/hooks/`:**
- Purpose: All stateful data logic; thin layer over Supabase + React Query
- Key files:
  - `use-links.ts` — CRUD for links + analytics record/query functions
  - `use-admin.ts` — admin-specific queries/mutations
  - `use-autosave.ts` — debounced save logic for editor
  - `use-editor-history.ts` — undo/redo stack
  - `use-plan-limits.ts` — derives current user's plan limits
  - `use-profile.ts` — user profile data
  - `use-user-role.ts` — admin role check
  - `use-device-fingerprint.ts` — passive device tracking

**`src/lib/`:**
- Purpose: Pure functions with zero React dependencies; safe to unit test in isolation
- Key files:
  - `link-mappers.ts` — `rowToSmartLink` / `smartLinkToRow` (sole DB↔domain mapping layer)
  - `block-utils.ts` — `createBlockDefaults(type)` factory
  - `slug-utils.ts` — `validateSlug`, `checkSlugAvailability`, `normalizeSlug`
  - `color-utils.ts` — background color extraction helpers
  - `protect.ts` — code protection initialization
  - `utils.ts` — Tailwind `cn()` helper

**`src/types/`:**
- Purpose: Domain type definitions, no runtime code
- Key files: `smart-link.ts` — defines `SmartLink`, `LinkBlock`, `SmartLinkButton`, `SubPage`, `BlockType`, and all effect/item interfaces

**`src/data/`:**
- Purpose: Hardcoded static configuration; not fetched from DB
- Key files:
  - `templates.ts` — `LinkTemplate[]` with pre-built link configurations and preview metadata
  - `plans.ts` — `PlanDefinition[]` with pricing, features, and limits UI copy

**`src/integrations/supabase/`:**
- Purpose: Isolates all Supabase SDK usage behind a single module
- `client.ts` exports the singleton `supabase` — import only from here
- `types.ts` is auto-generated by Supabase CLI; do not hand-edit

**`supabase/migrations/`:**
- Purpose: Ordered SQL schema migrations
- Naming: `{timestamp}_{uuid}.sql` (older) and `{timestamp}_{description}.sql` (newer)
- 20 migration files as of analysis date

**`supabase/functions/og/`:**
- Purpose: Deno-based Supabase Edge Function for OG image generation

## Key File Locations

**Entry Points:**
- `src/main.tsx`: React app bootstrap
- `src/App.tsx`: Route tree and provider composition
- `index.html`: SPA HTML shell (lang="pt-BR")

**Domain Model:**
- `src/types/smart-link.ts`: All TypeScript interfaces for SmartLink domain

**DB Mapping:**
- `src/lib/link-mappers.ts`: Only place that converts DB rows to/from domain objects

**Supabase Client:**
- `src/integrations/supabase/client.ts`: Import `supabase` from here exclusively

**Authentication:**
- `src/contexts/AuthContext.tsx`: Import `useAuth()` from here; never call `supabase.auth` directly in components

**Route Guards:**
- `src/components/ProtectedRoute.tsx`: Requires authenticated user
- `src/components/AdminRoute.tsx`: Requires authenticated user AND admin role

**Testing:**
- `src/test/`: All test files co-located in this directory
- `vitest.config.ts`: Test runner config

**Deployment:**
- `vercel.json`: SPA catch-all rewrite rule; all routes → `index.html`

## Naming Conventions

**Files:**
- Page components: PascalCase ending in `Page` or matching route name — `Dashboard.tsx`, `LinkEditor.tsx`, `PublicLinkPage.tsx`
- Admin pages: `Admin{Name}Page.tsx` — `AdminUsersPage.tsx`
- Hooks: kebab-case prefixed with `use-` — `use-links.ts`, `use-editor-history.ts`
- Lib utilities: kebab-case, descriptive — `link-mappers.ts`, `block-utils.ts`, `slug-utils.ts`
- UI primitives: kebab-case (shadcn convention) — `alert-dialog.tsx`, `scroll-area.tsx`
- Feature components: PascalCase — `SmartLinkPreview.tsx`, `BlockEditor.tsx`, `DashboardLayout.tsx`
- Effect components: PascalCase matching effect name — `SnowEffect.tsx`, `MatrixEffect.tsx`

**Directories:**
- Feature groupings: lowercase — `editor/`, `preview/`, `course/`, `support/`
- Admin section: `admin/` subdirectory under `pages/`

**TypeScript:**
- Interfaces: PascalCase — `SmartLink`, `LinkBlock`, `SmartLinkButton`
- Type aliases: PascalCase — `BlockType`, `LinkType`, `EntryAnimation`
- Hook return values: destructured object pattern — `{ data, isLoading }` from React Query, `{ state, set, undo, redo }` from custom hooks

## Where to Add New Code

**New page (protected route):**
- Create `src/pages/MyPage.tsx` as default export
- Add lazy import and `<Route>` in `src/App.tsx` wrapped in `<ProtectedRoute>`
- Add sidebar link in `src/components/AppSidebar.tsx`

**New admin page:**
- Create `src/pages/admin/Admin{Name}Page.tsx`
- Add lazy import and `<Route>` in `src/App.tsx` wrapped in `<AdminRoute>`

**New block type:**
1. Add string literal to `BlockType` union in `src/types/smart-link.ts`
2. Add fields to `LinkBlock` interface if needed in `src/types/smart-link.ts`
3. Add default values in `createBlockDefaults()` in `src/lib/block-utils.ts`
4. Add editor form case in `BlockEditor.tsx` (`src/components/editor/BlockEditor.tsx`)
5. Add render case in `BlockRenderer.tsx` (`src/components/preview/BlockRenderer.tsx`)
6. Add label entry to `BLOCK_LABELS` in `src/components/editor/BlockEditor.tsx`

**New data query/mutation:**
- Create a new custom hook in `src/hooks/use-{feature}.ts`
- Use `useQuery` / `useMutation` from `@tanstack/react-query`
- Import `supabase` from `src/integrations/supabase/client.ts`
- Import `useAuth` from `src/contexts/AuthContext.tsx`

**New utility function:**
- If no React: add to appropriate file in `src/lib/` or create `src/lib/{concern}-utils.ts`
- If React hook: add to `src/hooks/use-{feature}.ts`

**New UI primitive component:**
- Use `npx shadcn-ui@latest add <component>` — it will be placed in `src/components/ui/`
- Do not hand-write shadcn primitives

**New migration:**
- Create `supabase/migrations/{timestamp}_{description}.sql`
- Follow existing naming: `20260321000001_add_hero_image_height.sql`
